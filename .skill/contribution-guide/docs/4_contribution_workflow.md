# Contribution Workflow

Use the repository's own workflow instead of imposing a generic open-source process.

Checklist items to verify when the repository expects them:

- CLA or DCO steps are complete.
- CI passes locally or in the PR checks.
- Branch name follows the repo convention.
- Commits are signed or signed off when required.
- PR description matches the repository's review process.
- Documentation is updated when the repository expects it.

If the repository is small and direct, keep the workflow lightweight. If it has release gates or compliance checks, include them explicitly.
