# Repository Analysis

Before giving contribution advice, inspect the repository for the rules it actually uses. Do not assume every project has the same legal or workflow requirements.

Look for evidence in:

- README and CONTRIBUTING docs.
- PR templates and issue templates.
- CI workflows and status checks.
- Recent commit history and merge style.
- Branch protection rules or release scripts.

Infer the workflow from that evidence. If the repository does not clearly require something, say so instead of treating it as mandatory.
