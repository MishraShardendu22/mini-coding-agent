# Scoring and Output

Return a score out of 100 and explain it with repository evidence.

Recommended score buckets:

- 90-100: strong PR, consistent with the repository, low risk.
- 75-89: acceptable with minor issues or modest tradeoffs.
- 60-74: mixed quality, likely needs revision before merge.
- Below 60: significant mismatch with the repository or high regression risk.

The final report should include:

1. Summary
2. Problem solved
3. Strengths
4. Weaknesses
5. Required changes
6. Suggested improvements
7. Score table
8. Final verdict

Keep required changes separate from suggestions. Tie each score deduction to the repository's conventions, not to abstract ideals.
