# Contribution Rules

Group repository contribution requirements by the actual signals you find, not by generic Git advice.

Common repository-level requirements include:

- DCO sign-off.
- CLA acceptance.
- Signed commits.
- Branch naming rules.
- Commit message conventions.
- CI checks that gate PR readiness.
- Documentation or changelog updates.

Determine whether each rule is documented, enforced by CI, or only recommended. Only instruct the agent to enforce what the repository truly requires.
